### Java/Jakarta EE y .NET - Oposiciones TIC
Fecha de trabajo: 30/04/2026

Objetivo:
Cubrir Java/Jakarta EE y .NET con nivel suficiente para test y supuestos. No se trata de programar en examen, sino de entender arquitectura, componentes, persistencia, seguridad, despliegue e integracion.


============================================================
1. JAVA / JAKARTA EE - IDEA GENERAL
============================================================

Java:
- Lenguaje orientado a objetos.
- Independencia de plataforma mediante JVM.
- Tipado estatico.
- Ecosistema maduro para aplicaciones empresariales.

Jakarta EE:
- Evolucion de Java EE bajo Eclipse Foundation.
- Plataforma de especificaciones para aplicaciones empresariales.
- Orientada a aplicaciones web, servicios, persistencia, transacciones, seguridad e integracion.

Frase de examen:
"Java/Jakarta EE proporciona un ecosistema empresarial basado en especificaciones para construir aplicaciones web y servicios robustos, con componentes para persistencia, transacciones, seguridad e integracion."


============================================================
2. COMPONENTES JAVA/JAKARTA EE
============================================================

| Componente | Funcion |

============================================================
3. PERSISTENCIA EN JAVA
============================================================

JPA:
- Mapeo objeto-relacional.
- Entidades.
- EntityManager.
- JPQL.
- Relaciones.
- Transacciones.

Riesgos:
- N+1 queries.
- Transacciones mal delimitadas.
- Lazy loading mal gestionado.
- SQL injection si se construyen consultas inseguras.

Buenas practicas:
- Parametrizar consultas.
- Controlar transacciones.
- Diseñar modelo coherente.
- Medir rendimiento.


============================================================
4. SEGURIDAD EN JAVA/JAKARTA
============================================================

Controles:
- Autenticacion.
- Autorizacion por roles/atributos.
- Validacion de entrada.
- Proteccion CSRF/XSS.
- Gestion de sesiones.
- TLS.
- Gestion de secretos.
- Logs.

En Administracion:
- Integrar con identidad corporativa o sistemas admitidos.
- Trazabilidad.
- ENS.
- RGPD.


============================================================
5. .NET - IDEA GENERAL
============================================================

.NET:
- Plataforma de desarrollo de Microsoft multiplataforma en versiones modernas.
- Lenguaje habitual: C#.
- Usada para aplicaciones web, APIs, servicios, escritorio, nube y microservicios.

ASP.NET Core:
- Framework web moderno.
- APIs REST.
- Middleware.
- Inyeccion de dependencias.
- Configuracion.
- Autenticacion/autorizacion.

Frase de examen:
".NET, especialmente ASP.NET Core, permite construir aplicaciones web y APIs empresariales con soporte de inyeccion de dependencias, middleware, seguridad, persistencia, pruebas y despliegue en entornos modernos."

============================================================
7. PERSISTENCIA EN .NET
============================================================

Entity Framework Core:
- ORM.
- DbContext.
- Entidades.
- Migraciones.
- LINQ.
- Transacciones.

Riesgos:
- Consultas ineficientes.
- Carga excesiva.
- Migraciones no controladas.
- Inyeccion si se usa SQL bruto inseguro.


============================================================
8. JAVA VS .NET EN EXAMEN
============================================================

| Aspecto | Java/Jakarta EE | .NET |
|---|---|---|
| Lenguaje | Java | C# principalmente |
| Ejecucion | JVM | CLR/.NET runtime |
| Web/APIs | Servlets, JAX-RS, frameworks | ASP.NET Core |
| Persistencia | JPA | Entity Framework Core |
| Paquetes | Maven/Gradle | NuGet |
| Ecosistema | Multiplataforma, empresarial, abierto | Multiplataforma moderno, fuerte integracion Microsoft |

Respuesta madura:
No elegir por gusto personal. Elegir por requisitos, ecosistema existente, equipo, integracion, licencias, soporte, seguridad, mantenibilidad y estrategia tecnologica.
